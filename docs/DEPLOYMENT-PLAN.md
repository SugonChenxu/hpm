# Forge 部署推广规划（方案 A 定稿）

> 编写日期：2026-07-20 ｜ 状态：**决策已定（方案 A 自托管 / 局域网 / 同事免装 Node）**
> 背景：功能开发基本完成，目标从"个人自用"扩展到"推广给同事使用"。

---

## 0. 现状速览（部署相关，已核实）

| 项 | 现状 | 对推广的影响 |
|----|------|------|
| 架构 | Express 单进程托管 API + 前端静态文件（生产模式） | ✅ 单文件分发即可 |
| 监听 | `app.listen(3000)`，默认绑定所有网卡 | ✅ 局域网同网段可访问 |
| 数据库 | SQLite（`server/hpm.db`，**相对路径** `join(__dirname,"..","hpm.db")`） | ✅ 可整体拷贝迁移 |
| 前端资源 | `client/dist`（相对路径 `../../client/dist`） | ✅ 已预构建，分发无需 Vite |
| 鉴权 | **无** | ⚠️ 局域网内任何人可读写（见 §2-9） |
| 数据模型 | 所有表无 `user_id` | ✅ 自托管单用户天然适配 |
| 原生依赖 | `better-sqlite3@11`（预编译二进制） | ⚠️ 需随包附 node_modules |
| 构建依赖 | `vite@8`（需 Node ≥ 20.19 / 22） | ⚠️ 分发时**内置 Node**，同事免装 |
| **Mantis** | 路由齐全（`GET/PUT /mantis/connection`、`/sync`、`/projects`），前端故障管理页可填 Token | ✅ **已具备**，每人配自己的即可，无需改造 |
| **OA 物料导入** | Chrome 扩展/书签向 `localhost:3000` 注入 | ✅ 选装，开箱即用 |
| **腾讯会议 / PLM** | 适配器存在，但绑个人 OAuth / 未全面打通 | ❌ **不纳入推广范围** |

## 1. 部署形态（已决策：方案 A）

### 决策结论
- **方案 A 自托管**：每位同事在自己电脑上跑一份 Forge（各自独立数据库，数据互不可见）。
- **同一局域网**：同事之间在同一办公室网络，但各自实例独立。
- **同事免装 Node**：通过**绿色压缩包**分发（内置 Node 运行时 + node_modules + 预构建前端），双击脚本即可启动，无需安装 Node / npm / 构建。
- **外部对接范围**：Mantis = 必须（已具备）；OA = 选装（文档说明）；腾讯会议 / PLM = 不做。

### 分发模型（推荐）
```
Forge-绿色包/
├─ forge-node/            # 内置 Node 22 便携运行时（官方 zip 解压，无需安装）
├─ server/                # Express 服务 + node_modules（含 better-sqlite3 预编译）
│  ├─ src/index.js
│  └─ hpm.db              # 首次启动自动初始化（或附空模板库）
├─ client/dist/           # 已构建前端
├─ 启动.bat               # 双击：用包内 node 跑 server，绑定 0.0.0.0:3000，打印本机 LAN IP
├─ 开机自启.bat           # 可选：注册到「启动」文件夹 / HKCU\Run
├─ 备份.bat               # 拷贝 hpm.db
└─ 使用说明.txt           # Mantis 配置 / OA 选装 / 局域网访问 / 数据备份
```
同事流程：**解压 → 双击 `启动.bat` → 浏览器开 `http://localhost:3000`**。零门槛。

### 局域网共享变体（可选，需谨慎）
若希望"一个人主机、其他人浏览器访问其 IP"：
- 主机双击 `启动.bat`（已绑定 `0.0.0.0`），其他人访问 `http://<主机IP>:3000`。
- ⚠️ **无鉴权**：同网段任何人可读写主机全部数据。仅限高度信任的内网，且建议加一个简单访问口令（见 §3-P0 可选项）。
- 主机需保持开机；主机 IP 建议设静态或绑定主机名。

## 2. 更换电脑 / 给他人部署 —— 功能断点清单（按方案 A 复核）

> 方案 A 绿色包已解决大部分断点，下表标注每项状态。

1. **`better-sqlite3` 原生编译** → ✅ 已解决：随包附匹配 Node 22 的预编译 `node_modules`，同事不编译。
2. **Node 版本不符** → ✅ 已解决：包内 `forge-node/` 锁定 Node 22，启动脚本用包内 node。
3. **绝对路径写死** → ✅ 已解决：删 PM2，改用相对路径 `启动.bat`；服务器 db/dist 本就相对。
4. **PM2 开机自启注册表** → 🟡 改为：提供 `开机自启.bat`（注册「启动」文件夹快捷方式或 HKCU\Run 指向 `启动.bat`）。
5. **腾讯会议（M5）** → ❌ 不纳入推广（绑个人 OAuth）。绿色包中可禁用入口或保留但注明不可用。
6. **Mantis 连接** → ✅ 已具备：前端故障管理页可填地址+Token，存 `mantis_connection` 单行，自托管天然适配。仅需**文档化配置步骤**。
7. **OA 物料导入** → ✅ 选装：Chrome 扩展向 `localhost:3000` 注入，同事本机即自己实例，开箱可用。文档说明即可。
8. **PWA / Service Worker** → 🟡 小注意：以 `localhost` 访问 PWA 正常；若用 IP 访问，SW scope 不同，旧缓存需清理 Site Data。
9. **无鉴权红线** → ⚠️ 局域网共享变体下暴露风险；默认自托管（本机访问）无此问题。共享变体建议加访问口令。
10. **单用户数据模型** → ✅ 自托管场景无需改动。

## 3. 方案 A 落地步骤（绿色包）

### P0 — 制作绿色分发包（必做，先于分发）
- [ ] **`启动.bat`**：`cd` 到包目录 → 用 `forge-node/node.exe server/src/index.js` → 绑定 `0.0.0.0:3000` → 启动后打印本机局域网 IP 与访问地址（便于同事访问主机）。
- [ ] **内置 Node**：下载官方 Node 22 Windows zip，解压为 `forge-node/`，验证 `forge-node/node.exe --version`。
- [ ] **随包附 `node_modules`**：在 Node 22 环境下 `npm install`（server），确保 `better-sqlite3` 预编译匹配。
- [ ] **预构建前端**：`client/dist/` 已构建（当前生产包即此状态）。
- [ ] **`开机自启.bat`**（可选）：创建「启动」文件夹快捷方式或写 HKCU\Run → `启动.bat`（隐藏窗口）。
- [ ] **`备份.bat`**：拷贝 `server/hpm.db`（含 `-wal`/`-shm`）到带时间戳备份目录。
- [ ] **`使用说明.txt`**：安装步骤、Mantis 配置（故障管理页 → 设置 → 填地址/Token → 测试连接）、OA 选装指引、局域网访问说明、数据备份与升级方法。
- [ ] **升级流程文档**：覆盖 `server/` + `client/dist/`，保留 `hpm.db` 不动；或提供 `升级.bat` 自动备份后覆盖。
- [ ] **可选安全项**：局域网共享变体下，加一个简单访问口令（前端登录遮罩 + 后端校验 env `FORGE_ACCESS_TOKEN`），低工作量，非强制。
- [ ] **（验证）路径审计**：确认仓库内无残留 `D:/HPM` 等绝对路径（`ecosystem.config.js` / `start.sh` 仅你本机用，不入绿色包）。

### P1 — 同事安装流程（极简）
- [ ] 收到 `Forge-绿色包.zip` → 解压到任意目录（如 `D:\Forge`）。
- [ ] 双击 `启动.bat` → 浏览器自动/手动开 `http://localhost:3000`。
- [ ] （必须）故障管理页配置自己的 Mantis 地址 + Token，点"测试连接"验证。
- [ ] （选装）OA 物料导入：安装 Chrome 扩展/书签脚本。
- [ ] （可选）`开机自启.bat` 注册开机启动；浏览器"安装"为 PWA 桌面图标。

### P2 — 配套与运维
- [ ] 数据备份脚本定时化（同事自行或你提供计划任务）。
- [ ] 给非技术同事的一页纸图文手册。
- [ ] 你侧维护：改代码后重新 `build` + 打包，发布新版本 zip（Git 仓库保留源码，Release 附绿色包）。

## 6. 验证方法（制作完成后如何自查）

> 结论：**不强制需要另一台电脑**。同一台机器 + 不同目录即可完成功能验证；局域网绑定用手机/同 WiFi 设备即可验证；真正"另一台电脑"是黄金标准但非必需。

### 验证层级

| 层级 | 是否需要另一台设备 | 验证什么 | 怎么做 |
|------|:---:|------|------|
| L1 功能验证（同机异目录） | ❌ 不需要 | 相对路径、内置 Node、免安装、数据持久化 | 把绿色包解压到**非 D:/HPM** 的目录（如 `D:\Forge-Test`），双击 `启动.bat` |
| L2 局域网绑定 | 🟡 手机/同 WiFi 设备（或本机 curl 自身 LAN IP） | 服务监听 `0.0.0.0`、同事可访问 | 手机浏览器开 `http://<电脑LAN_IP>:3000` |
| L3 真机验收 | ✅ 另一台电脑/VM | 完整同事视角 | 拷贝包到另一台电脑，重复 L1 |

### L1 同机验证步骤（最常用）
1. **停掉当前 PM2 实例**（避免端口 3000 冲突）：`bash start.sh stop`（或在 `启动.bat` 支持 `PORT` 环境变量，测试时 `set PORT=3001` 即可共存，无需停 PM2）。
2. 把绿色包解压到 `D:\Forge-Test`（**关键：必须不是 D:/HPM**，才能证明没有绝对路径写死）。
3. 双击 `启动.bat` → 观察窗口打印的 Node 版本（应等于内置 Node 22，而非你全局装的）和"访问地址 http://localhost:3000"。
4. 浏览器开 `http://localhost:3000`（建议**无痕窗口**，模拟同事干净环境，避开本机 PWA/localStorage 干扰）。
5. **功能抽查**：新建一个项目 → 刷新页面 → 重启 `启动.bat` → 数据仍在（证明 SQLite 落在包目录、持久化正常）。
6. **Mantis**：故障管理页 → 设置 → 填你的 Mantis 地址+Token → 测试连接（需真实凭证，验证端到端打通）。
7. **OA（选装）**：装 Chrome 扩展，试一次导入，确认指向 `localhost:3000` 正常。
8. 验证完删掉 `D:\Forge-Test` 即可，不影响你正在用的 D:/HPM 开发环境。

### L2 局域网绑定验证（可选但建议）
- 同机执行：`curl http://<你的LAN_IP>:3000`（LAN IP 在 `启动.bat` 启动时会打印）→ 返回 HTTP 200 即绑定成功。
- 或用**手机**连同一 WiFi，浏览器开 `http://<电脑LAN_IP>:3000` → 能打开即同事也能访问。
- 若打不开：检查 Windows 防火墙是否放行 3000 入站（绿色包可附一条"防火墙放行.bat"）。

### 判定通过标准
- [ ] 非 D:/HPM 目录可独立启动，不依赖全局 Node / PM2 / D:/HPM
- [ ] 页面正常渲染、功能可用、重启后数据不丢
- [ ] Mantis 测试连接成功
- [ ] （如需共享）手机/同网段可访问
