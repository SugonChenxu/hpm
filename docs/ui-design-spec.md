## 主题色系统

> 来源：`architecture.md` §8.6

| 语义 | 色值 | MUI Token | 用途 |
|------|------|-----------|------|
| 主色 | `#1565C0` | `primary.main` | 按钮、链接、选中态、进行中状态、P2 优先级 |
| 成功绿 | `#2E7D32` | `success.main` | 已完成状态、正常进度、门禁通过 |
| 警告橙 | `#ED6C02` | `warning.main` | 临期警告（<2周）、P1 优先级、3天内到期 |
| 错误红 | `#D32F2F` | `error.main` | 已逾期、P0 优先级、门禁未通过、DI 超标 |
| 灰色系 | MUI `grey` palette | — | 未开始、空状态、辅助文字、分割线 |

**优先级色标**：P0 = `#D32F2F` / P1 = `#ED6C02` / P2 = `#1565C0`

### 1.2 顶部操作栏

| 区域 | 组件 | 说明 |
|------|------|------|
| 左侧 | `Button` (contained, primary) | 「+ 新建项目」，点击跳转 `/projects/new` |
| 右侧 | `IconButton` × 2 | 视图切换图标（网格/列表）、设置齿轮图标 |
| 间距 | — | 左右两端对齐，`px: 3`, `py: 2` |

### 1.3 筛选栏（FilterBar）

| 控件 | MUI 组件 | 行为 |
|------|----------|------|
| 状态下拉 | `Select` | 选项：全部 / 进行中 / 已结项 / 已归档；默认「全部」 |
| 类别下拉 | `Select` | 选项：全部 + 9 种项目类别；默认「全部」 |
| 搜索框 | `TextField` + `InputAdornment`(🔍) | 实时过滤（300ms debounce），匹配 `code` 和 `name` |
| 统计徽标 | `Chip` × 3 | 分别显示「全部 N」「进行中 N」「高风险 N」，点击触发对应筛选 |

筛选栏整体使用 `Paper` 包裹，`elevation={0}`，`variant="outlined"`，与下方内容区有 16px 间距。

### 1.4 统计卡片行（StatsBar）

四张 `Card` 水平等宽排列，每张含：

| 字段 | 说明 |
|------|------|
| 图标 | MUI Icon（如 `FolderOpen`, `PlayCircle`, `Warning`, `ErrorOutline`） |
| 数值 | 大号数字（`Typography variant="h4"`），颜色对应语义色 |
| 标签 | 小号灰色文字：「项目总数」「进行中」「高风险」「逾期待办」 |

**数据来源**：`GET /api/projects`（统计） + `GET /api/tasks/overdue`（逾期待办数）

### 1.5 项目卡片（ProjectCard）

每张卡片为一个 `Card` 组件（`elevation={2}`，hover 时 `elevation={4}`），点击跳转 `/projects/:id`。

#### 卡片内容布局

```
┌─────────────────────────────┐
│  [类别标签 Chip]             │  ← 左上角，9 种类别小标签
│                             │
│  HG4-001          ← 代号    │  ← 左上主体，Typography h6, bold
│  银河4号智能网关   ← 名称    │  ← 代号下方，body2, grey
│                             │
│  ┌─────────────────────┐    │
│  │ ████████████░░░░░░░ │    │  ← 进度条 (LinearProgress)
│  │   65%               │    │
│  └─────────────────────┘    │
│                             │
│  当前: M3 研发测试           │  ← 当前阶段名 (body2)
│                             │
│  ┌─────┐                    │
│  │ ●正常│ ← 风险色标        │  ← 右下角，圆点 + 文字
│  └─────┘                    │    绿=正常 / 橙=有风险 / 红=已逾期
│  更新: 07-07 14:30          │  ← 左下角，caption, grey
└─────────────────────────────┘
```

#### 卡片字段细则

| 字段 | 组件 | 样式 |
|------|------|------|
| 类别标签 | `Chip` size="small" | 9 种类别各有预设色（新品=蓝, OEM=绿, 定制=紫, …） |
| 代号 | `Typography` variant="h6" | `fontWeight: 600` |
| 名称 | `Typography` variant="body2" | `color: text.secondary` |
| 进度条 | `LinearProgress` | `variant="determinate"`, 颜色跟随风险色标 |
| 进度百分比 | `Typography` variant="caption" | 进度条右侧或下方 |
| 当前阶段 | `Typography` variant="body2" | 格式「当前: {phase.name}」 |
| 风险色标 | `Chip` + 圆点 | 圆点直径 10px，绿/橙/红 |
| 更新时间 | `Typography` variant="caption" | `color: text.disabled`，格式 `dayjs(updated_at).format('MM-DD HH:mm')` |

#### 风险判定规则

| 风险等级 | 颜色 | 条件 |
|----------|------|------|
| 正常 | `#2E7D32` 绿 | 所有阶段未逾期，且最近阶段距计划结束 > 14 天 |
| 有风险 | `#ED6C02` 橙 | 最近进行中阶段距计划结束 ≤ 14 天 |
| 已逾期 | `#D32F2F` 红 | 存在任一阶段 status = '已逾期' |

#### 点击行为

整张卡片可点击 → `navigate(/projects/${id})`

### 1.6 空状态（Empty State）

当筛选后无项目时，显示：

```
┌─────────────────────────────────────────┐
│                                         │
│            [📁 FolderOpen Icon]         │  ← MUI Icon, 64px, grey
│                                         │
│          还没有项目，创建第一个吧        │  ← Typography h6, grey
│                                         │
│          [+ 新建项目]  Button           │  ← contained primary
│                                         │
└─────────────────────────────────────────┘
```

- 如果是因为筛选导致为空：文字改为「没有匹配的项目，试试调整筛选条件」，按钮改为「清除筛选」
- 全局无项目时：居中大空状态 + 引导按钮

### 1.7 加载状态

- **首次加载**：4 列骨架屏（`Skeleton` 卡片 × 8），模拟卡片高度约 200px
- **筛选刷新**：卡片区域短暂 fade 过渡（不显示骨架屏，避免闪烁）
- **错误状态**：`Alert` severity="error" + 重试按钮

### 1.8 响应式断点

| 断点 | 宽度 | 列数 | 卡片间距 |
|------|------|:--:|------|
| `xs` | < 600px | 1 列 | 16px |
| `sm` | 600–900px | 2 列 | 16px |
| `md` | 900–1200px | 3 列 | 20px |
| `lg` | ≥ 1200px | 4 列 | 24px |

使用 MUI `Grid container` + `Grid item xs={12} sm={6} md={4} lg={3}` 实现。

StatsBar 在 xs 断点下改为 2×2 网格；FilterBar 在 sm 以下改为垂直堆叠。

### 1.9 组件清单

| 组件名 | 来源（架构文档 §4.2） | 类型 |
|--------|----------------------|------|
| `FilterBar` | `shared/FilterBar.jsx` | 复用 |
| `StatsBar` | DashboardPage 内联 | 页面级 |
| `ProjectCardGrid` | DashboardPage 内联 | 页面级 |
| `ProjectCard` | `project/ProjectCard.jsx` | 项目组件 |

### 2.3 阶段时间线（PhaseTimeline）— 纵向地铁图

#### 视觉设计

使用 MUI `Timeline` 组件（`@mui/lab/Timeline`），纵向排列。每个节点为一个阶段（PHASE）或门禁（GATE）。

#### 节点类型与样式

| 节点类型 | 图标 | 颜色 | 连接线 |
|----------|------|------|--------|
| 已完成阶段 | `CheckCircle` (实心勾) | `#2E7D32` 绿 | 绿色实线 |
| 进行中阶段 | `PlayCircle` (带脉冲动画) | `#1565C0` 蓝 | 蓝色实线（到下一节点变灰） |
| 未开始阶段 | `RadioButtonUnchecked` (空心圆) | `grey.400` | 灰色虚线 |
| 已逾期阶段 | `Error` (感叹号) | `#D32F2F` 红 | 红色实线 |
| 门禁-已通过 | `Verified` (盾牌勾) | `#2E7D32` 绿 | 绿色实线 |
| 门禁-待检查 | `GppMaybe` (盾牌叹号) | `#ED6C02` 橙 | 橙色虚线 |
| 门禁-未通过 | `Dangerous` (盾牌叉) | `#D32F2F` 红 | 红色实线 |
| 门禁-未激活 | `OutlinedFlag` | `grey.400` | 灰色虚线 |

#### 每个 PHASE 节点内容

```
┌──────────────────────────────┐
│  ● M3 研发测试    ← 阶段名   │  ← Typography subtitle1, fontWeight 600
│    07-15 ~ 08-30  ← 计划日期 │  ← Typography caption, grey
│    已用 23/46 天   ← 进度     │  ← LinearProgress 小型
│    ████████░░░░░░  50%       │
│                              │
│    ⚠ TR2 待检查  ← 门禁预览  │  ← 仅当该阶段出口有门禁时显示
└──────────────────────────────┘
```

当前进行中的阶段节点使用 **脉冲动画**：`@keyframes pulse { 0% { box-shadow: 0 0 0 0 rgba(21,101,192,0.4); } 70% { box-shadow: 0 0 0 10px rgba(21,101,192,0); } 100% { box-shadow: 0 0 0 0 rgba(21,101,192,0); } }`，动画无限循环。

#### 门禁节点内容

```
┌──────────────────────────────┐
│  ◆ TR2 技术评审              │
│  阈值 DI < 130               │
│  当前 DI: 45 ✓              │  ← 绿色=达标 / 红色=超标
│  通过时间: 07-14 16:00       │  ← 仅已通过时显示
└──────────────────────────────┘
```

#### 交互行为

- **点击阶段节点** → 右侧内容区切换为该阶段的详细信息（Tab 内容联动）
- **点击门禁节点** → 右侧内容区展开 GateCheckPanel
- 当前选中节点使用 `primary.main` 背景高亮（`bgcolor: rgba(21,101,192,0.08)`）
- 使用 MUI Timeline 的 `position="right"` 模式，内容显示在节点右侧

### 2.4 右侧内容区（阶段联动）

#### DI 值比对视觉规则

| 条件 | 进度条颜色 | Alert 类型 |
|------|-----------|------------|
| `current_di ≤ threshold` | `#2E7D32` 绿 | `severity="success"` |
| `current_di > threshold` | `#D32F2F` 红 | `severity="error"` |

#### 手动通过按钮行为

- 条件全部满足时：`Button variant="contained" color="primary"`，可点击
- 条件未全部满足时：`disabled`，`Tooltip` 显示「请先满足所有门禁条件」
- 点击后弹出 `ConfirmDialog`：「确认手动通过 TR2 门禁？通过后将自动推进到下一阶段。」
- 确认后调用 `POST /api/projects/:id/gates/:gid/pass`，成功后刷新时间线和面板

### 2.6 空状态与异常

### 3.6 快速添加（Quick Add）

每列底部内联添加区域：

```

│                      │

│ │                   │ │
```

**交互流程**：

3. 按 Esc 或点击外部 → 折叠回收，放弃输入
4. 添加成功后新卡片有一个短暂的「入场动画」（`fadeIn + slideDown`，300ms）

## 附录：通用交互约定

### A. 加载状态

| 场景 | 组件 | 说明 |
|------|------|------|
| 整页首次加载 | `CircularProgress` 居中 | 页面级 loading |
| 局部数据刷新 | `Skeleton` 对应形状 | 卡片/列表/时间线 |
| API 操作中 | 按钮 `loading` spinner | 保存/删除/通过等 |
| 乐观更新 | 无 loading | 拖拽操作即时反馈 |

### B. 错误处理

| 场景 | 处理 |
|------|------|
| 网络错误 | `Alert severity="error"` + 重试按钮，错误信息来自 API `error` 字段 |
| 404 | 专用 404 页面或 Alert + 返回 |
| 表单校验 | MUI `TextField error` + `helperText`，按字段提示 |

### C. 通知

使用 MUI `Snackbar`（非 `Alert`），位置 `anchorOrigin={{ vertical: 'bottom', horizontal: 'center' }}`，`autoHideDuration={3000}`。

| 操作 | 通知文案 |
|------|----------|
| 创建成功 | 「项目/任务创建成功」 |
| 更新成功 | 「已保存」 |
| 删除成功 | 「已删除」+ Undo 按钮（可选 P1） |
| 拖拽流转 | 无通知（即时视觉反馈即可） |
| 门禁通过 | 「TR2 门禁已通过，阶段已推进至 M4」 |

### D. 确认对话框（ConfirmDialog）

统一使用 `shared/ConfirmDialog.jsx`：

| 属性 | 说明 |
|------|------|
| `title` | 对话框标题 |
| `content` | 确认文案 |
| `confirmText` | 确认按钮文字（默认「确认」） |
| `confirmColor` | 确认按钮颜色（`error` 用于删除操作） |
| `onConfirm` | 确认回调 |
