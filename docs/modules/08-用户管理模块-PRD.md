# 用户管理模块 PRD（系统设置）

> 模块编号：M8（系统设置 / 横切功能，独立于 M1–M7 业务模块）
> 创建日期：2026-07-21
> 状态：✅ 已实装并部署

## 1. 模块定位
面向内网多用户集中部署场景的系统级管理功能。将「加用户 / 改密码」从命令行直连数据库操作，迁移为网页界面操作，降低运维门槛、避免手动跑库出错。

## 2. 用户与权限模型
- 多用户、各自独立管理；业务数据按 `owner_id` 隔离（见顶层 PRD 与 MEMORY）。
- 内网小团队、人人平等：**无角色权限**——任何已登录用户均可进入用户管理页进行账号管理。
- 认证：服务器会话（`express-session` + 文件存储），有效期 30 天，满足「登录尽量长」。
- 账号开通：管理员（当前为部署者）手动逐个添加，不开放公开注册。

## 3. 功能清单
| 功能 | 说明 | 入口 |
|---|---|---|
| 用户列表 | 查看全部账号（用户名、创建时间） | 用户管理页 `/users` |
| 新增账号 | 手动添加同事账号（用户名 + 初始密码） | 用户管理页「＋ 新增用户」 |
| 修改自己的密码 | 输入原密码 + 新密码（自助） | 用户管理页「🔑 修改我的密码」/ 顶栏「修改密码」 |
| 重置他人密码 | 管理员直接重设某用户密码（无需原密码） | 列表行「重置密码」 |
| 删除账号 | 删除指定用户；**禁止删除当前登录账号** | 列表行「删除」（自己置灰） |

## 4. 接口规范
| 方法 | 路径 | 说明 | 保护 |
|---|---|---|---|
| GET | /api/users | 用户列表（不含密码哈希） | 需登录 |
| POST | /api/users | 新增用户 `{username, password}` | 需登录 |
| POST | /api/users/me/password | 改自己密码 `{oldPassword, newPassword}` | 需登录（校验原密码） |
| PUT | /api/users/:id/password | 重置他人密码 `{password}` | 需登录 |
| DELETE | /api/users/:id | 删除用户（禁删自己） | 需登录 |

## 5. 业务规则
- 密码强度：≥ 6 位。
- 用户名唯一（`UNIQUE` 约束，重复返回 409）。
- 删除用户为物理删除；其历史业务数据（项目/任务/故障/会议/周报等）仍按原 `owner_id` 保留，变为「无主」状态，**不做级联清理**（后续可升级为「停用」软删 + `is_active` 字段）。
- 不能删除当前登录账号（避免锁死），后端显式 400。

## 6. 前端实现
- 路由：`/users` → `UserManagementPage`
- 侧边栏：「系统设置」分组 → 用户管理（内联 `PeopleIcon`，规避 `@mui/icons-material` 的 Vite 8 兼容问题）
- 顶栏：登录用户区「🔑 修改密码」按钮，复用共享组件 `ChangePasswordDialog`
- 新增 / 重置 / 删除均带二次确认与 Snackbar 反馈；对自己行禁用删除并提示
