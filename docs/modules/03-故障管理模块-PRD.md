## 二、模块职责（增量变更说明）

`[增量变更]` 在 v1.0 "双数据源融合 + DI 计算" 基础上，新增三条核心职责：

1. **可视化分析**：DI 趋势折线图 + 缺陷分类柱状图，仅展示有效数据（DI > 0 / 缺陷数 > 0），支持缩放和 tooltip
2. **自动报告生成**：按固定模板生成故障数据报告，一键复制至剪贴板
3. **多项目上下文切换**：切换 Mantis 项目时图表与报告联动刷新

`[保持不变]` 原有 Mantis 数据拉取、本地缺陷管理、DI 计算、缺陷列表/看板、状态流转、Mantis 推送等职责不变。

## 三、用户故事

### US1 — DI 趋势洞察
> As a **项目经理**，I want 在项目详情页查看 DI 趋势折线图（仅展示 DI > 0 的历史节点），so that 我能快速判断项目质量是向好还是恶化。

**验收标准**：
- 横轴 = 时间（按同步记录的时间戳），纵轴 = DI 数值
- DI = 0 的数据点自动过滤，不在图表中渲染
- 支持缩放（鼠标滚轮/手势）和悬浮 tooltip（显示精确日期 + DI 值）
- 切换项目时图表自动刷新

### US2 — 缺陷分类分布
> As a **项目经理**，I want 查看按分类（BIOS/BMC/HW/Perf）的缺陷分布柱状图，so that 我能快速定位哪个子系统问题最多。

**验收标准**：
- X 轴 = 分类维度（BIOS / BMC / HW / Perf），Y 轴 = 缺陷条数
- DI = 0 或缺陷数为 0 的分类条目自动隐藏（不渲染该柱子）
- 柱子颜色区分，tooltip 展示分类名称 + 精确数量
- 切换项目时同步刷新

### US3 — 一键周报
> As a **项目经理**，I want 系统根据当前项目数据自动生成标准格式的故障报告，并一键复制全文，so that 我能直接粘贴到周报/邮件中，省去手动统计的时间。

**验收标准**：
- 报告模板：
  ```
  BUG情况：
  项目BUG状况：当前项目DI=XX、BUG=XX条、已解决=XX条，解决率=XX%
  遗留BUG xx条：BIOS-xx、BMC-xx、HW-xx、Pef-xx
  ```
- "一键复制"按钮，点击后将报告全文写入系统剪贴板
- 复制成功后给予 toast 提示（如"已复制"）
- 切换项目时报告内容联动刷新

### US4 — 全局统计概览
> As a **项目经理**，I want 在模块顶部看到当前项目的故障总数量、已解决数量、解决率，so that 我无需手动计算即可掌握整体状况。

**验收标准**：
- 三个指标卡片横向排列：故障总数 / 已解决 / 解决率（百分比，保留 1 位小数）
- 数据来源于 Mantis 拉取 + 本地缺陷的合并结果
- 切换项目时自动刷新
- 无数据时显示 "暂无数据"（非 0/0/0%）

### US5 — 健壮性与体验
> As a **日常用户**，I want 系统在网络异常或鉴权失败时给出明确提示，且数据刷新后页面不跳转，so that 我的操作流不被中断。

**验收标准**：
- 鉴权失败：提示"Mantis 鉴权失败，请检查 API Token 配置"
- 网络超时：提示"请求超时，请检查网络后重试"，并提供"重试"按钮
- 无项目数据：提示"当前账号下暂无 Mantis 项目数据"
- DI 全为 0：正常渲染空图 + 提示"DI 值均为 0，无趋势数据可展示"
- 手动刷新/自动同步后列表滚动位置锁定，不跳转至顶部
- 手动刷新按钮醒目可见，刷新时显示 loading 状态

## 八、架构注意事项增量

`[保持不变]` Adapter 模式、冲突处理、DI 计算缓存机制不变。

`[增量变更]`：

- **缓存策略**：前端请求先查 `sync_cache`（TTL 5min），命中直接返回；未命中/过期则透传至 MantisAdapter 拉取并回填缓存。手动刷新走 `Cache-Control: no-cache` 跳过缓存层。
- **图表性能**：recharts `<Brush>` 组件处理缩放；大数据量（>500 数据点）时前端做降采样（LTTB 算法）。
- **请求去重**：同一项目同一接口的并发请求合并为单次调用，避免重复访问 Mantis API。
