# M6 — 周报模块详细 PRD

> **关联顶层 PRD**: `docs/PRD.md` v2.0  
> **开发优先级**: ⑥（最后开发——数据聚合层，依赖 M1–M5）  
> **依赖**: M1(项目进度) + M2(待办事项) + M3(故障管理) + M4(物料管理) + M5(会议纪要)

---

## 一、模块职责

**独立生成各项目专属总结周报**。按自然周（周一至周日）聚合各模块数据——进度摘要、任务完成情况、故障统计、物料到货状态、会议纪要清单——以结构化模板输出可读的周报内容。用户可编辑润色后存档或分享。

---

## 二、功能清单

### 2.1 周报生成

| 功能 | 描述 |
|------|------|
| **选择周范围** | 日历组件选择自然周（周一→周日，默认本周）；也可快速切换「上周」「本周」 |
| **选择项目** | 单项目周报（默认）或「全部项目汇总」 |
| **一键生成** | 点击「生成周报」——系统自动聚合 M1–M5 数据，按模板填充各板块内容 |
| **生成进度** | 各模块数据加载时显示板块级进度条（进度→待办→故障→物料→会议→完成） |

### 2.2 周报模板结构

周报按以下固定结构组织，每板块数据自动填充：

```
━━━ 项目周报：{项目代号} - {项目名称} ━━━
报告周期：YYYY-MM-DD ~ YYYY-MM-DD

一、项目进度
  - 当前阶段：{阶段名称}（{计划开始} ~ {计划结束}）
  - 阶段进度：{N}/{M} 阶段已完成（{百分比}%）
  - 本周完成：{阶段A完成转段}/{阶段B开始}
  - 下周计划：{阶段名} 启动 / {阶段名} 继续推进
  - 风险提示：{如有逾期阶段在此列出}

二、待办事项
  - 本周新增：{N} 项
  - 本周完成：{N} 项
  - 本周逾期：{N} 项
  - 重点待办（P0）：{逐条列出}
  - 下周关键任务：{从进行中/待开始的P0任务提取}

三、故障缺陷
  - 当前 DI 值：{XX.XX}（阶段阈值：{XX.XX}，{达标/未达标}）
  - 本周新增：Critical {N} / Major {N} / Minor {N}
  - 本周关闭：{N} 项
  - 未关闭总数：{N} 项
  - 重点故障（Critical & Major）：{逐条列出}

四、物料状态
  - 本周到货：{N} 项
  - 在途物料：{N} 项
  - 逾期末到货：{N} 项（{逐条列出料号+名称+逾期天数}）
  - 待下单物料：{N} 项

五、会议纪要
  - 本周会议：{N} 场
  - 关键决议：{逐条列出未完成的 action items}
  - 下周会议安排：{列出已计划的下周会议}

六、下周重点计划
  （用户手动填写）
```

### 2.3 周报编辑与管理

| 功能 | 描述 |
|------|------|
| **内容编辑** | 生成后所有板块内容可编辑（Markdown），用户可增删改内容、调整表述 |
| **手动补充** | 「六、下周重点计划」板块完全手动填写；其他板块可在自动生成基础上修改 |
| **保存草稿** | 支持保存为草稿，后续继续编辑 |
| **版本历史** | 每次保存生成新版本，可查看历史版本（类似 Git diff） |
| **周报列表** | 按项目查看历史周报列表——周范围/生成时间/状态(草稿/已定稿) |

### 2.4 数据聚合逻辑

| 板块 | 数据来源 | 聚合方式 |
|------|---------|---------|
| 项目进度 | M1 phases + gates | 查询阶段状态变更记录（actual_start/actual_end 在本周范围内）+ 当前阶段信息 |
| 待办事项 | M2 tasks | 查询 created_at/updated_at 在本周范围内的任务；按状态分计入「新增」「完成」；查询 due_date < now 且 status != completed 计入「逾期」 |
| 故障缺陷 | M3 issues | 按项目+阶段统计未关闭 DI 汇总；本周新增按 created_at >= week_start；本周关闭按 status=已关闭且 updated_at 在本周范围 |
| 物料状态 | M4 materials | 查询 actual_delivery 在本周范围的计入「本周到货」；status=在途计入「在途」；status=已逾期计入「逾期末到货」；status=待下单计入「待下单」 |
| 会议纪要 | M5 meetings | 查询 start_time 在本周范围的计入「本周会议」；关联的 action_items WHERE status=待处理 计入「关键决议」 |

## 三、数据模型

### 3.1 `weekly_reports`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | — |
| project_id | INTEGER FK | 关联项目（NULL=全部项目汇总） |
| week_start | TEXT | 周起始日期（周一） |
| week_end | TEXT | 周结束日期（周日） |
| title | TEXT | 自动生成：{项目代号} 周报 {YYYY年第N周} |
| content_json | TEXT | JSON——六个板块的结构化内容（模板结构） |
| status | TEXT | 草稿 / 已定稿 |
| version | INTEGER | 版本号（每次保存递增） |
| created_at | TEXT | — |
| updated_at | TEXT | — |

## 四、接口

| Method | Path | 说明 |
|--------|------|------|
| POST | /api/weekly-reports/generate | 生成周报——传入 {project_id, week_start, week_end} → 返回聚合后的 JSON |
| GET | /api/weekly-reports | 周报列表（?project_id=&status=） |
| GET | /api/weekly-reports/:id | 查看周报详情 + 内容 |
| PUT | /api/weekly-reports/:id | 更新周报（编辑后保存，自动递增 version） |
| GET | /api/weekly-reports/:id/versions | 查看版本历史 |
| GET | /api/weekly-reports/:id/versions/:version | 查看指定版本内容 |
| GET | /api/weekly-reports/aggregate | 周报聚合数据预览接口——直接返回各板块原始数据（不生成模板文本） |

## 五、架构注意事项

- **只读聚合**：周报模块不修改 M1–M5 任何数据，仅做聚合读取
- **生成性能**：生成周报涉及 5 个模块的查询——首次实现用同步串行查询（数据量小够用），后续如果慢可改为 Promise.all 并行
- **模板引擎**：周报模板文本生成逻辑独立封装——`WeeklyReportTemplateEngine { renderProgress(), renderTasks(), renderIssues(), renderMaterials(), renderMeetings() }` ——方便后续调整模板格式
- **孙氏对齐**：自然周始终以周一为起始日，周日为结束日；前端使用 ISO week date 计算
