## 修复：物料页白屏 React #130（根因 Vite 8 + @mui/icons-material CJS 不兼容）
- **初步误判**：升级 x-date-pickers 到 v7.29.4 无效
- **最终根因**：Vite 8/rolldown 对 `@mui/icons-material` v5.18 CJS 的 `exports.default` 导出解析异常，`import Icon from "@mui/icons-material/.."` 在运行时变成 module namespace object 而非 React 组件 → #130。仅物料页使用了 icon 并实际渲染（其他页只 import 未用 → tree-shake → 不触发）。
- **定位手段**：增量排除法——极简组件验证所有 import 正常 → 逐行加 JSX → 定位到 SortArrow 组件里的 KeyboardArrowUp/DownIcon → 验证 icon 文件存在但运行时被 Vite 8 错误打包。
- **修复**：所有 `@mui/icons-material` 替换为 Unicode（▲▼▾📥＋↓✕🔍↩），同时加 `client/vite.config.js`（alias 三处 x-date-pickers 子路径→具体.js 防患于未然）。
- **产物**：MaterialListPage.jsx 完整重写（表格/排序/内联编辑/状态标签/列宽拖拽/批量操作/导入导出/撤销导入），puppeteer 验证 rootHtmlLen=17712 whiteScreen=false。
