## 修复：物料页选择项目后白屏（React #130）
- 根因：`@mui/x-date-pickers` v6.20.2 无 `exports` 字段，在 Vite 8 下子路径目录导入（LocalizationProvider/AdapterDayjs/DatePicker）被解析成 object 而非组件 → React #130 → 整页白屏。物料页独有（唯一用 DatePicker 的页面）。
- 修复：升级 `@mui/x-date-pickers` 到 v7.29.4（`^7.29.4`），恢复官方推荐的三行子路径导入。v7 各子路径目录带正确 package.json(module/main)，Vite 8 可正确解析。
- 诊断手段：puppeteer-core + 系统 Chrome 无头复现 + dev 模式未压缩 React 拿到完整 componentStack（脚本在 scripts/pptr/）。
- 重新 build（vite 8.1.3 通过）→ 生产 forge 重启。

## 运维：网页打不开（生产 forge 丢失，dev 残留占用）
- 现象：诊断白屏时切到 dev 模式（forge-api:3001 + forge-ui:5173），生产 forge(3000) 被停；之后 3000 打不开。
- 修复：`pm2 delete all` 清掉 dev 残留 → `pm2 start ecosystem.config.js` 启动生产单进程 forge(3000) → `pm2 save` 固化 dump.pm2。
- 教训：dev 诊断完必须切回生产并 `pm2 save`，否则 resurrect 会拉起 dev 残留，导致 3000 打不开。
