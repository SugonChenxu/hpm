## BugFix: PM2 开机自启固化（彻底解决重启后服务丢失）

### 问题根因
- Windows 上 `pm2 startup` 报错 "Init system not found"（PM2 原生不支持 Windows 开机自启）
- 重启后 PM2 daemon 不启动，进程列表丢失，服务全部挂掉
- 之前的 PM2 配置只有崩溃自动重启（autorestart），没有开机自启

### 修复方案
- **注册表 HKCU\Run** 注册 `HPM-PM2-AutoStart` → 指向 `D:\HPM\scripts\pm2-resurrect.bat`
- **bat 脚本**：延迟 15 秒（等系统就绪）→ 用完整 node 路径执行 `pm2 resurrect`
- **pm2 save**：进程列表保存到 `C:\Users\chenxu\.pm2\dump.pm2`
- **备用 VBS**：`scripts/pm2-autostart.vbs`（隐藏窗口版，当前未启用）

### 完整链路
```
开机登录 → 注册表 HKCU\Run 触发 → pm2-resurrect.bat
→ 延迟15秒 → node pm2 resurrect → 从 dump.pm2 恢复 hpm-server + hpm-client
```

### 崩溃恢复（已有）
- PM2 `autorestart: true` — 进程崩溃后自动重启
- `max_memory_restart: 512M` — 内存超限自动重启

### 文件
- `scripts/pm2-resurrect.bat` — 主启动脚本（注册表使用）
- `scripts/pm2-autostart.vbs` — 备用方案（VBS隐藏窗口版）
- `start.sh` — 更新说明，标注开机自启已配置
- 注册表：`HKCU\Software\Microsoft\Windows\CurrentVersion\Run\HPM-PM2-AutoStart`

### 验证
- PM2 状态：hpm-server + hpm-client 均 online
- HTTP 检查：Server(3001)=200, Client(5173)=200
- dump.pm2 文件存在（1.5MB），pm2 save 成功

### 安全策略适配
- `wscript.exe` 被 LOLBin 安全策略拦截 → 改用 bat 文件方案
- bat 路径全 ASCII，内容全 ASCII，无安全问题
