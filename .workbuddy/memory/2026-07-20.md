# 2026-07-20 工作日志

## 会议计划-输出物：周期项模型重构 + 视觉修复

### 修复的三个问题
1. **自动编号**：由「中间小号数字(width:14,0.6rem)」改为「左置前缀 `N.`，字号 0.72rem 与文字一致，加粗」，更连贯美观。
2. **周期标志**：由「标题与 ▶ 之间的小 chip」改为「标题前的连贯前缀小标签（每周/隔周/每月，带浅色底）」，不再挤压文字。
3. **设周期后无法勾选（操作失败）**：根因 `request()` 返回 `{ok,data}`，但 `handleToggleOutput` 周期分支误用 `created.id`（undefined）→ `update(undefined)` 404 抛错。

### 架构重构（关键）
- `meeting_outputs` 表新增 `is_template`(0/1) 与 `source_id` 两列；历史带 cycle 的行规整为 `is_template=1`。
- 区分「周期模板」(`is_template=1`，定义，不直接展示) 与「每周实例」(`is_template=0`，携带 cycle 供展示，source_id 指向模板)。
- 后端 GET：`outputs` 仅取 `is_template=0`；周期模板按周/隔周/每月规则生成虚拟项 `recurring_<id>`(带 `source_id`)。
- 新增 `POST /week-meetings/outputs/cycle-instance`：按 `(week_key,weekday,title)` upsert 每周实例（已存在则仅更新 is_done），避免重复行；完成态仅代表当前周期。
- `handleSetCycle`：虚拟项→更新原模板；普通项→升级为模板(`is_template=1`)。
- `handleDeleteOutput`：虚拟项删模板(停后续周期)；实例/一次性删该条。
- `handleEditOutput`：周期实例同步改模板标题。
