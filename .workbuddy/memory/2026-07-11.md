## PLM 排程真实接口逆向（2026-07-11 用户 F12 抓包 + 主理人实测）
- **关键发现：排程数据接口 = `POST /3dspace/common/emxFreezePaneGetData.jsp`**（非 REST，是 v6 jsp servlet）。用户 F12 抓到真实请求，并发来「Copy as curl」。
- **必带鉴权/头**（缺则失败）：
  - Cookie：`JSESSIONID=...cas01`（CAS SSO 会话，会过期需重贴）+ `testcookie=1` + `afs=<uuid>`
  - **CSRF Token**：请求头 `ENO_CSRF_TOKEN: R70H-SY1E-...-IMPP`，且配套 `csrfTokenName: ENO_CSRF_TOKEN`（缺则 PLM 拒写/异常）
  - `Origin` / `Referer` 必须同源 `https://plm.sugon.com`（Referer 是完整 `emxIndentedTable.jsp?...` 长 URL，含 tableMenu=PMCWBSTableMenu、portal=PMCSchedule、mode=PMCWBS、suiteKey=ProgramCentral、parentOID、objectId 等表定义参数）
  - `Content-Type: text/plain`（用户 curl 显示 undefined，实测 text/plain 可）
- **请求参数**（query string）：`fpTimeStamp=<ms>`、`objectId=<项目OID 点分格式 如 19520.49557.60552.29879>`、`firstTime=true`、`toolbarData=expandLevel=1|emxExpandFilter=1`、`IsStructureCompare=`、`sortColumnName=PhaseEstimatedStartDate`、`sortDirection=ascending`。**POST body 为空（Content-Length:0）**。
- **实测结果**：
  - 仅 GET / 缺 CSRF → HTML 500 错误页（`内部服务器错误`）。
  - 带齐 Cookie+CSRF+POST 原样重放 → 返回 **XML**（`<mxRoot><action>error</action><message>java.lang.NullPointerException</message>`），栈在 `com.matrixone.apps.framework.ui.UITableIndented.getXMLByOID(...:2702)`。说明**认证/接口/路径全对**，但 `getXMLByOID` 因缺某前置参数/会话状态 NPE。
  - 直接 `GET emxIndentedTable.jsp` 也 **HTTP 500 空响应** → 证实 PLM 排程页**强会话状态依赖**（须先在浏览器完成导航建立 session 表格配置，才能取数；无法用裸 curl 独立 bootstrap）。
- **推论 / 待解**：
  - 读路径：因 session 状态依赖，HPM 独立 HTTP 取数需复刻浏览器完整导航序列（或抓到 getData 的真实 Response 结构）。
  - **写路径（P1 核心目标）尚未探明**：需用户在 PLM 里真实"新增/编辑一条 WBS 任务"，用 F12 抓该动作的「Copy as curl」→ 这才是 P1 要复刻的写接口与字段映射。
- **下一步（已向用户要）**：
  - ① 用户在浏览器 Network「Response」标签复制 `emxFreezePaneGetData.jsp` 的**真实返回 XML**（含任务行/字段）→ 直接给出字段映射，无需我复刻请求。
  - ② 用户在 PLM 新增/编辑一条 WBS 任务 → 「Copy as curl」发我 → 拿到写接口与参数。
